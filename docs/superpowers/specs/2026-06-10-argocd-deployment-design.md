# GMVIS — ArgoCD Deployment Design

**Date:** 2026-06-10
**Status:** Approved
**Reference implementation:** [bcewhub/bcewhub-app](https://github.com/bcewhub/bcewhub-app) (Helm chart + SealedSecrets + GitHub Actions → ArgoCD)

## Goal

Deploy the GMVIS site (static pages + admin API) to the existing talos
Kubernetes cluster via ArgoCD, with release-driven CI/CD from GitHub Actions,
following the same architecture and operational pattern as `bcewhub-app`.

## Decisions made

| Decision | Choice |
|---|---|
| Target cluster | Same talos cluster as bcewhub (ingress-nginx, cert-manager `letsencrypt-prod`, sealed-secrets, `nfs-client` StorageClass, ArgoCD already installed) |
| Public host | `gmvis.org` (held in `values-prod.yaml`; one-line change if the domain differs) |
| Deploy trigger | GitHub Release published → build + deploy. `workflow_dispatch` for manual runs. Push to `main` does not deploy. |
| Image architecture | **Single image.** `server/Dockerfile` already bakes the static site into `/site` and the Express app serves static + `/api` + `/media`. The separate nginx static image is dropped. |
| Namespace | `prod-gmvis` |
| ArgoCD app name | `gmvis-prod` |
| Image name | `ghcr.io/borisnan99/gmvis-web` |
| Repo access for ArgoCD | HTTPS, no credentials (repo is public) |

the single image makes the nginx static image unused.

## Helm chart

All templates are ports of the existing, already-hardened `k8s/` manifests.

- **Deployment** — 1 replica, `strategy: Recreate` (SQLite is a single-writer
  on a ReadWriteOnce volume; Recreate prevents two pods sharing it).
  Existing securityContext preserved: runAsNonRoot uid/gid/fsGroup 1000,
  seccomp RuntimeDefault, readOnlyRootFilesystem, drop ALL capabilities,
  `/tmp` emptyDir. Readiness + liveness probes on `/api/health`. Env:
  `PORT`, `DATA_DIR=/data`, `SITE_DIR=/site`, `NODE_ENV=production`,
  `COOKIE_SECURE=true`, plus `ADMIN_PASSWORD` and `SESSION_SECRET` from the
  `gmvis-app` secret. `imagePullSecrets: ghcr-pull`.
  `image.tag` is required — the chart fails to render if it is unset
  (`required` helper), so a deploy can never silently ship `latest`.
- **Service** — ClusterIP, port 3001 (`http`).
- **Ingress** — single backend (the app serves everything). Host from
  `values-prod.yaml`. Annotations: `cert-manager.io/cluster-issuer:
  letsencrypt-prod`, `nginx.ingress.kubernetes.io/proxy-body-size: "200m"`
  (video uploads), `nginx.ingress.kubernetes.io/ssl-redirect: "true"`, and
  the cluster's `cloudflare.com/dns-enabled` DNS annotation. TLS secret
  `gmvis-web-tls`.
- **PVC** — `gmvis-api-data`, 5 Gi, `storageClassName: nfs-client`,
  ReadWriteOnce. Annotated `argocd.argoproj.io/sync-options: Prune=false`
  so ArgoCD auto-prune can never delete the database/uploads volume.
- **SealedSecrets** — committed encrypted, sealed against the cluster
  controller's public cert
  (`~/dev/cititech/talos-cluster/terraform/k8s-base/sealed-secrets/pub-cert.pem`,
  overridable via `SEALED_SECRETS_CERT`):
  - `gmvis-app` (Opaque): `ADMIN_PASSWORD`, `SESSION_SECRET`
  - `ghcr-pull` (dockerconfigjson): fine-grained GHCR read-only PAT

## CI workflow (`.github/workflows/deploy.yml`)

**Triggers:** `release: types [published]`; `workflow_dispatch` with inputs
`ref` (default `main`) and `deploy` (boolean, default true).

**Job `build-push`:**
1. Checkout the requested ref.
2. Log in to GHCR with `GITHUB_TOKEN` (`packages: write`).
3. Compute tags: always `sha-<12-char-short-sha>`; on release also
   `<release-tag>` and `latest`; on dispatch also `manual-<run_number>`.
4. `docker build -f server/Dockerfile .` from the repo root; push all tags.
5. Output `image_tag` (`sha-…`) for the deploy job.

**Job `deploy`** — runs on release, or dispatch with `deploy == true`.
Concurrency group `deploy-gmvis-prod`, `cancel-in-progress: false`.

1. Install argocd CLI **pinned to v3.4.2** (version bcewhub validated;
   bump deliberately after smoke-testing).
2. All argocd calls wrapped in a retry helper (5 attempts, 15 s apart).
3. `argocd login $ARGOCD_SERVER --grpc-web` using repo secrets
   `ARGOCD_SERVER`, `ARGOCD_USERNAME`, `ARGOCD_PASSWORD`
   (dedicated `gmvis-ci` local account, RBAC-scoped to `gmvis-prod`).
4. `argocd app create gmvis-prod --upsert` —
   repo `https://github.com/borisnan99/GMVIS.git`,
   path `deploy/helm/gmvis`, revision = the deployed ref/tag,
   dest `https://kubernetes.default.svc` ns `prod-gmvis`,
   `--sync-option CreateNamespace=true --sync-option ServerSideApply=true`,
   `--sync-policy automated --auto-prune --self-heal`,
   `--values values-prod.yaml`, `--helm-set image.tag=$IMAGE_TAG`.
5. `argocd app set gmvis-prod -p image.tag=$IMAGE_TAG`.
6. `argocd app get gmvis-prod --refresh` (register the OutOfSync), then
   `argocd app wait gmvis-prod --sync --health --operation --timeout 600`.

**No explicit `argocd app sync`.** With automated sync policy the `app set`
already changes desired state and the controller starts the sync itself; a
manual sync races it ("another operation is already in progress"). One
syncer only: ArgoCD owns the sync, CI moves the tag and waits.

## Secrets inventory

| Secret | Where | Purpose |
|---|---|---|
| `ARGOCD_SERVER` / `ARGOCD_USERNAME` / `ARGOCD_PASSWORD` | GitHub Actions repo secrets | CI → ArgoCD login |
| `gmvis-app` (SealedSecret in chart) | cluster `prod-gmvis` | `ADMIN_PASSWORD`, `SESSION_SECRET` for the admin portal |
| `ghcr-pull` (SealedSecret in chart) | cluster `prod-gmvis` | image pull auth for `ghcr.io` |

## Operator runbook (`docs/deployment-runbook.md`)

One-off bootstrap, adapted from bcewhub's:

1. Create the `gmvis-ci` ArgoCD local account, set its password, grant it
   `app:gmvis-prod` RBAC; add the three GitHub repo secrets.
2. Generate a fine-grained GHCR read-only PAT; seal as `ghcr-pull`.
3. Generate `ADMIN_PASSWORD` + `SESSION_SECRET` (`openssl rand`); seal as
   `gmvis-app`.
4. Verify `helm template` renders with `values-prod.yaml`.
5. Commit sealed secrets + values; push to `main`.
6. DNS: if `gmvis.org` is not managed by the cluster's Cloudflare
   integration, create the record manually pointing at the cluster ingress.
7. Cut release `v0.1.0` → watch the Deploy workflow → ArgoCD goes green.
8. Smoke test: `https://gmvis.org/api/health` returns `{"ok":true}`; admin
   login works; upload one image; restart the pod and confirm the post and
   image survive (PVC persistence check).

## Error handling

- Transient ArgoCD RPC failures: retried 5× in CI.
- Missing `image.tag`: chart render fails fast (required value).
- Deploy never overlaps another deploy (concurrency group).
- Data loss via prune: prevented by `Prune=false` on the PVC.
- Incident override: `argocd app set gmvis-prod --sync-policy none`, do the
  manual work, re-enable; the next CI run re-asserts automated sync anyway.

## Testing / verification

- `helm template gmvis-prod deploy/helm/gmvis --values
  deploy/helm/gmvis/values-prod.yaml --set image.tag=test` renders cleanly;
  rendering **without** `image.tag` must fail.
- `kubectl apply --dry-run=client -f` (or `kubeconform`) over the rendered
  output.
- Existing Playwright suite (119 tests) untouched — no application code
  changes in this work.
- End-to-end: first release deploys green via `argocd app wait`; smoke
  tests in runbook §8.

## Out of scope

- Staging/preview environments (bcewhub has them; GMVIS starts prod-only —
  the values-file layout leaves room to add `values-staging.yaml` later).
- CI test workflow (running Playwright in Actions) — separate piece of work.
- Backups of the SQLite volume (worth doing later; bcewhub's
  `backup-cronjob.yaml` is a ready template).
